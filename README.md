# Corner Grocer Item Tracking Program

Project Summary

The Corner Grocer Item Tracking Program was created to help a grocery store analyze customer purchasing patterns by tracking how often each grocery item is purchased throughout the day. The program reads grocery purchase data from an input file, counts the frequency of each item, allows users to search for specific items, displays all purchase frequencies, generates a histogram, and creates a backup file containing the frequency data.

What I Did Particularly Well

I designed the program using object-oriented programming principles by separating responsibilities between the `GroceryItem` and `GroceryTracker` classes. This made the code easier to organize, understand, and maintain. I also implemented file input and output, a menu-driven interface, and input validation to improve the overall user experience.

Where I Could Enhance My Code

I could improve the program by making the search feature case-insensitive so users could find items regardless of how they type the name. I could also improve the input validation by handling unexpected input more gracefully and providing clearer error messages. Another improvement would be to reduce duplicate code by creating helper functions for repeated tasks, making the program easier to maintain and update in the future.

 Challenges

The most challenging part of this project was designing the classes and implementing the logic to track item frequencies while avoiding duplicate entries. I overcame this by breaking the problem into smaller pieces, testing each function individually, reviewing course materials, and using the C++ documentation as a reference.

 Transferable Skills

This project strengthened my understanding of object-oriented programming, file processing, data structures, debugging, and writing modular code. These skills will transfer directly to future software development projects and other programming courses.

Maintainability, Readability, and Adaptability

I made the program maintainable by organizing the code into separate classes with clearly defined responsibilities. I used descriptive variable and function names, consistent formatting, comments where appropriate, and modular functions. This structure makes it easier to update or expand the program with additional features in the future.
